**Key Technical Concepts**
- **UI/Workflow Refactoring:** The core coaching interface was fundamentally changed from a single-page athlete-centric model to a two-level hierarchy (Athlete Overview -> School Pod) to better represent the recruiting process.
- **Component Scoping:** Existing React components (modals, sidebars) were made reusable in different contexts by adding props (, ) that conditionally change their behavior and API endpoints.
- **On-Demand vs. Cached Data:** A hybrid approach to data fetching was implemented. High-focus views (like a single School Pod) trigger a fresh re-computation of metrics, while list views read from a cache for performance, with a manual refresh option.
- **Root Cause Analysis:** Successfully diagnosed and fixed bugs deep in the backend business logic (e.g., the issue lifecycle cooldown) that were causing inconsistent UI states.
