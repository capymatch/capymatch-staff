All Users

The entry point to CapyMatch. Supports Sign In and Create Account tabs. Three demo accounts are displayed for quick access. JWT tokens are issued on successful login and stored in the AuthContext for all subsequent API requests.

Accept Invite Page

/accept-invite/:token
Invited Coaches

When a director invites a coach, they receive a unique link with a token. This page validates the token, displays the invitation details, and allows the coach to set their password and create their account. The invite is marked as "accepted" and the coach is immediately logged in.

Token-based invite validation
Set password + auto-login on accept
Shows inviter name and team assignment
Graceful error for expired/invalid tokens

Invite Coaches

/invites
Director Only

Directors manage their coaching staff here. Create invitations by entering an email, name, and optional team assignment. Invites are sent via Resend email API. The page shows delivery status (sent/failed/pending), allows resending, and provides a copy-link fallback for when email delivery fails (sandbox mode).

Create invite with name, email, team
Resend email delivery with status tracking
Copy invite link fallback
Resend count and expiry display

Architecture & Tech Stack

Frontend

React 18 with JavaScript. Tailwind CSS for styling. Shadcn/UI component library. React Router for navigation. Axios for API calls with JWT auth headers. AuthContext for global authentication state.

Backend

FastAPI (Python) with modular routers. JWT auth middleware on all routes. Motor async MongoDB driver. Service layer for ownership, AI, email, and startup logic. 30+ API endpoints across 12 routers.

Database

MongoDB with collections: users, athletes, invites, events, event_notes, recommendations, actions, assignments, messages, reassignment_log. All queries exclude _id from responses.

AI Layer

GPT 5.2 via emergentintegrations library. 9 AI features across V1 and V2. 45s timeout with 1 retry. Structured output parsing. Every suggestion includes WHY + EVIDENCE + OWNER + CTA.

Authentication

JWT (PyJWT) with bcrypt password hashing. Two roles: director (global access) and coach (scoped to assigned athletes). Invite-based onboarding for coaches. Director self-registration blocked.

Data Ownership

primary_coach_id links athletes to coaches. In-memory ownership cache refreshed on startup and after reassignments. Directors see all; coaches see only their athletes across every mode.
