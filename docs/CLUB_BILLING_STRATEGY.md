# CapyMatch Club Billing — Packaging Strategy

## A. Packaging Philosophy

| Plan | Theme | What it delivers |
|------|-------|-----------------|
| **Starter** ($199/mo) | Visibility | See your program. Know your athletes. Start tracking. |
| **Growth** ($329/mo) | Coordination | Inbox, events, advocacy — stay on top of the pipeline. |
| **Club Pro** ($449/mo) | Operating System | Full workflow: approvals, AI, analytics, autopilot. |
| **Elite** ($699/mo) | Intelligence + Oversight | Digest, loop insights, unlimited AI, program brief. |
| **Enterprise** (custom) | Control + Customization | Admin, integrations, multi-location, API. |

**Core principle:** Every tier is *coherent and useful*. Lower tiers are not broken versions of higher tiers — they're intentionally scoped experiences. Upgrade pressure comes from *seeing what's possible*, not from hitting walls.

## C. Plan-by-Plan Matrix

### Starter ($199/mo · 25 athletes · 3 coaches)
**The director can:** See Mission Control KPIs, manage athlete roster and coach assignments, deep-dive into Support Pods and School Pods, receive notifications, message coaches.

**Limited:** Needs Attention shows top 3 only. No inbox, no outbox, no events, no advocacy, no AI, no analytics.

**Why this works:** Starter is genuinely useful for a small club just getting organized. The director can see their program, assign coaches, and track athletes. The absence of Inbox and Events creates natural upgrade pressure as the club grows.

### Growth ($329/mo · 50 athletes · 6 coaches)
**Adds:** Director Inbox (full risk feed), Review Requests, Event creation + prep, Advocacy (20 recs/month), School Relationships, Recruiting Signals (view only), full Needs Attention.

**Limited:** No Live Event Mode, no Pipeline Escalations, no AI, no Program Intelligence, no Autopilot.

**Why this works:** Growth gives the director coordination tools. They can see who needs attention, create events, build advocacy recommendations. The absence of Live Mode and AI creates upgrade pressure toward Club Pro.

### Club Pro ($449/mo · 75 athletes · 10 coaches) ← *Recommended*
**Adds:** Pipeline Escalations, Live Event Mode, Unlimited Advocacy, Program Intelligence + Trends, AI (50/month), Coach Watch AI, Autopilot, Coach Health, CSV Export, Trend Data.

**Limited:** No Weekly Digest, no Loop Insights, no AI Program Brief, no unlimited AI.

**Why this works:** Club Pro is the "real product." This is where CapyMatch becomes an operating system. Directors get the full workflow loop: inbox → action → escalation → autopilot → analytics. The jump from Growth to Pro is the biggest value inflection.

### Elite ($699/mo · 125 athletes · 20 coaches)
**Adds:** Unlimited AI, AI Program Brief, Weekly Digest, Loop Insights, Full Trend Analytics, Priority Support.

**Why this works:** Elite is for large, serious clubs. The digest and insights features give the director a 30,000ft view they can't get in Pro. Unlimited AI removes friction for heavy users.

### Enterprise (Custom · Unlimited)
**Adds:** Admin panel, Custom integrations, Multi-location, API access, Dedicated support, Unlimited everything.

**Why this works:** Enterprise is for multi-location organizations and franchises. The admin panel, integrations, and API unlock programmatic control.

## E. Never-Gated Features (Product Integrity)

These features MUST remain available on all plans. Gating them would make the product feel broken:

1. **Mission Control KPIs** — The dashboard must always load. A blank dashboard = broken product.
2. **Athlete List** — Directors must always see their athletes.
3. **Support Pods** — The core athlete deep-dive. Blocking this breaks the product's value proposition.
4. **School Pods (basic)** — Basic school view must work.
5. **Roster Assignments** — Directors must be able to assign coaches.
6. **Notifications** — Blocking notifications breaks communication.
7. **Messaging** — Core communication channel.
8. **Profile Management** — Basic self-service.
9. **Login / Auth** — Obviously.

## G. Implementation Approach

### Backend Architecture
- **`club_plans.py`** — Plan definitions, feature entitlements matrix, gating functions
- **`routers/club_plans.py`** — API endpoints: list plans, get current, check feature, set plan, bulk entitlements
- **`club_subscriptions` collection** — MongoDB: org_id, plan_id, status, billing_cycle, dates

### Frontend Architecture
- **`PlanContext.js`** — React context that hydrates entitlements on login, exposes `can(featureId)` and `getAccess(featureId)`
- **`PlanGate.js`** — Wrapper component: renders children if feature allowed, else renders UpgradePrompt
- **`UpgradePrompt.js`** — Reusable upgrade card with plan-specific accent colors

### How It Works (No Messy Conditionals)
```jsx
// Instead of:
{user.plan === "club_pro" || user.plan === "elite" ? <CoachHealth /> : <Locked />}

// We do:
<PlanGate feature="coach_health" name="Coach Health" minPlan="club_pro">
  <CoachHealth />
</PlanGate>
```

The `PlanGate` component reads from `PlanContext`, which fetched entitlements once on login. Zero conditional rendering spaghetti.

### Feature Flag Structure
```json
{
  "plan_id": "club_pro",
  "entitlements": {
    "director_inbox": { "allowed": true, "access": true },
    "ai_email_draft": { "allowed": true, "access": 50 },
    "weekly_digest": { "allowed": false, "access": false }
  }
}
```

## H. V1 Rollout Recommendation

### Gate Now (V1)
- Director Inbox (→ Growth)
- Director Outbox / Review Requests (→ Growth)
- AI Program Brief (→ Elite)
- Coach Health (→ Club Pro)
- Recruiting Signals (→ Growth)

### Leave Ungated (V1)
- Events (gate after Stripe integration is live)
- Advocacy (gate after usage tracking is implemented)
- AI features (gate after usage metering is built)
- Program Intelligence (gate after usage tracking)

### Reason
Gate the *visible, non-destructive* features first. These are sections on Mission Control that can cleanly show an UpgradePrompt without breaking any workflow. Usage-based limits (AI drafts, advocacy recs) require a metering system that should be built in V2.

### V2 Additions
- Stripe integration for real billing
- Usage metering (AI drafts, advocacy recs)
- Event and Advocacy page-level gates
- Soft limit banners (approaching athlete/coach caps)
- Annual billing toggle on billing page
