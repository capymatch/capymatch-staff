**Key Technical Concepts**
-   **Backend for Frontend (BFF):** The creation of the  endpoint is a prime example. The backend now performs the complex logic of computing the relationship state, allowing the frontend () to be a simple, stateless presentation component.
-   **State-Driven UI:** The UI (summaries, CTAs) dynamically changes based on a calculated state from the backend (e.g., No Signals Yet, Hot Opportunity, Stalled), ensuring consistency.
-   **Weighted Scoring & Time Decay:** A deterministic model in  that assigns weights to different engagement signals (replies, clicks, views) and decays their value over time to calculate a current, relevant interest score.
-   **Finite State Machine (FSM):** The 10-state matrix implemented in the backend acts as a state machine, transitioning the relationship status based on the score, signal types, and time since last interaction.
